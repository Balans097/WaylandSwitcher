# Установка Wayland Switcher на Fedora Linux (GNOME, Wayland)

## Требования

- Fedora 44+, рабочий стол GNOME 50+ на Wayland
- Права `root` (все команды ниже выполняются через `sudo`)
- Nim 2.x и компилятор C (`gcc`) — только для сборки из исходников

---


## Клавиши управления и коррекции раскладки
**PAUSE** — исправление раскладки последнего набранного слова из всей фразы (до ближайшего левого пробела)

**Shift + PAUSE** — исправление раскладки всей набранной фразы

**Shift (с удержанием) + PAUSE** — исправление раскладки всей набранной фразы + смена регистра символов

---



## Сборка из исходников

```bash
nim c --threads:on -d:release WaylandSwitcher.nim
```

Результат — исполняемый файл `WaylandSwitcher` в текущей директории.

---

## Установка бинарного файла

Программа должна находиться в стандартном системном пути, чтобы SELinux разрешил
её запуск от имени системного сервиса. Рекомендуется `/usr/bin/`:

```bash
sudo cp WaylandSwitcher /usr/bin/wayland-switcher
sudo chmod +x /usr/bin/wayland-switcher
```

---

## Установка systemd-сервиса

WaylandSwitcher умеет установить unit-файл самостоятельно:

```bash
sudo wayland-switcher -i
```

Команда создаёт файл `/lib/systemd/system/wayland-switcher.service` и прописывает
в него путь к текущему исполняемому файлу. Если установка прошла успешно, вы
увидите сообщение:

```
WaylandSwitcher daemon successfully installed, version: 0.5
```

### Альтернатива: вручную

Если потребуется отредактировать unit-файл, его полный путь:
`/lib/systemd/system/wayland-switcher.service`

Содержимое, которое записывает `-i`:

```ini
[Unit]
Description=WaylandSwitcher - keyboard layout switcher
Requires=local-fs.target
After=local-fs.target
StartLimitIntervalSec=10
StartLimitBurst=3

[Service]
Type=simple
ExecStart=/usr/bin/wayland-switcher -r
#User=wayland-switcher
Restart=on-failure
RestartSec=3

[Install]
WantedBy=sysinit.target
```

## Настройка

Запустите интерактивный конфигуратор:

```bash
sudo wayland-switcher -c
```

Конфигуратор последовательно попросит:

1. **Нажать Enter** — для автоматического определения клавиатуры среди устройств `/dev/input/event*`
2. **Нажать клавишу (или комбинацию)** переключения раскладки в вашей системе
3. **Нажать клавишу** для исправления ввода (по умолчанию рекомендуется Pause/Break)

Готовый конфиг сохраняется в `/etc/wayland-switcher/default.conf`.
Откройте его для дополнительной настройки:

```bash
sudo nano /etc/wayland-switcher/default.conf
```

Доступные параметры:

| Параметр             | Описание                                                                 | По умолчанию        |
|----------------------|--------------------------------------------------------------------------|---------------------|
| `keyboard`           | Путь к устройству клавиатуры                                             | определяется автоматически |
| `mouse`              | Путь к устройству мыши                                                   | `/dev/input/mice`   |
| `layout-switch-key`  | Сканкод клавиши или комбинации переключения раскладки (например `29+42`) | определяется автоматически |
| `replace-key`        | Сканкод клавиши исправления ввода (например `119` — Pause/Break)        | определяется автоматически |
| `reverse-mode`       | `true` — Pause исправляет всю фразу, Shift+Pause — только слово         | `false`             |
| `delay`              | Задержка между событиями (мс). Увеличьте при ошибках вывода             | `10`                |

Чтобы узнать сканкод нужной клавиши:

```bash
sudo showkey
```

---

## Включение и запуск

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now wayland-switcher.service
```

---

## Проверка работы

```bash
# Состояние сервиса
sudo systemctl status wayland-switcher.service

# Логи в реальном времени
journalctl -u wayland-switcher.service -f

# Отладочный запуск в терминале (подробный вывод всех событий)
sudo wayland-switcher -d
```

---


## Удаление сервиса

```bash
sudo systemctl stop wayland-switcher.service
sudo systemctl disable wayland-switcher.service
sudo wayland-switcher -u
```

## ⚠️ Особенности Fedora 44

### SELinux

В Fedora по умолчанию включён SELinux. Бинарник обязательно должен находиться в `/usr/bin/` (или другом стандартном системном пути) — именно поэтому установка в `/home/` или произвольную директорию работать не будет: контекст `user_home_t` не разрешает запуск от имени системных служб.

Если сервис не запускается, проверьте контекст SELinux:

```bash
ls -Z /usr/bin/wayland-switcher
```

Должно быть `system_u:object_r:bin_t:s0`. Если контекст отличается — восстановите его:

```bash
sudo restorecon -v /usr/bin/wayland-switcher
```

Если проблема сохраняется, установите утилиты управления политиками и явно назначьте тип `bin_t`:

```bash
sudo dnf install -y policycoreutils-python-utils
sudo semanage fcontext -a -t bin_t "/usr/bin/wayland-switcher"
sudo restorecon -v /usr/bin/wayland-switcher
sudo systemctl restart wayland-switcher.service
```

Посмотреть, блокирует ли SELinux запуск:

```bash
sudo ausearch -m avc -ts recent | grep wayland-switcher
```

### Доступ к `/dev/uinput` и `/dev/input/event*`

WaylandSwitcher читает события клавиатуры напрямую из ядра и создаёт виртуальное
устройство через `/dev/uinput`. В Fedora оба устройства по умолчанию доступны
только `root`, поэтому сервис запускается от `root` (см. unit-файл выше).

### Совместимость с GNOME на Wayland

WaylandSwitcher работает на уровне ядра (`/dev/input`) и не зависит от оконной
системы. Однако тестирование приложения будет осуществляться только на Wayland и корректная работа приложения в других оконных системах не гарантируется.

> **Известное ограничение:** WaylandSwitcher несовместим с ремапперами клавиш
> типа `keyd`. Если вы используете `keyd`, отключите его перед запуском.
> Для переключения раскладки рекомендуется назначить **одиночную** клавишу
> (без модификаторов) в настройках GNOME.
